###建造者模式/生成器模式

将一个复杂对象的构建与它的表示分离，使得同样的构建过程可以创建不同的表示

在建造者模式中有四个角色：
    1. produce产品类
        模板方法和基本方法

    2. Builder抽象建造者
        规范产品的组建，由子类实现

    3. ConcreteBuilder具体建造者
        实现抽象类定义的所有方法，并返回一个组建好的对象

    4. Director导演类
        负责安排已有模板的顺序


使用场景：
    1. 相同的方法，不同的执行顺序，产生不同结果，可以用建造者模式
    2. 多个部件或零件，都可以装配到一个对象中，但产生的结果又不相同，可以用建造者模式



虽然与工厂方法模式同为创建类模式，但注重点不同
